Chrome DevTools Extension

See the network graph behind any website

NetPhantom visualises every request a page makes as a live force-directed graph, a geolocated world map, and a filterable request table — all inside Chrome DevTools. No data ever leaves your browser.

Install extension View on GitHub

Everything you need to understand network traffic

All views are live, interactive, and filter each other.

◈

3D Force Graph

Nodes are origins, edges are cross-origin calls. Orbit in 3D, zoom, drag nodes. Click a node to filter all other views to that domain.

◉

World Map

Server IPs are geolocated to country-level dots with arced edges from the page origin. Hover for hostname and request stats. Click to filter.

≡

Request Table

Filter by request type (Fetch/XHR, JS, CSS, Image, WebSocket, …) or by URL substring. Click any row to open the detail sidebar.

⊞

Detail Sidebar

Per-request tabs: query params, request and response headers, timing waterfall, and full TLS/SSL certificate details.

✦

AI Insights

Automatically flags anomalies: rate limit responses, slow endpoints, unusually large payloads, and high server error rates.

↓

HAR Export

Export the full captured session as a standard .har file — compatible with any HAR viewer or performance tool.

⬡

Request Mocking

Intercept any request by URL pattern and return a custom status code, headers, and JSON body without touching the server.

⏸

Session History

Each page navigation creates a snapshot tab. Switch between them to compare traffic across navigations in the same session.

How it works

Two independent capture paths feed a single panel through the service worker.

Browser tab
  └─ content/interceptor.js     wraps fetch / XHR / WebSocket
                                fires CONTENT_REQUEST → service worker (real-time)

DevTools page  (devtools/devtools.js)
  └─ chrome.devtools.network    onRequestFinished → full headers, timing, TLS, serverIP
  └─ chrome.devtools.network    onNavigated → triggers per-page snapshot

Service worker  (background/service-worker.js)
  ├─ chrome.webRequest          caches real HTTP status codes (fixes devtools status:0 bug)
  ├─ port devtools-{tabId}      receives events from devtools.js; buffers up to 300
  ├─ port panel-{tabId}         flushes buffer on connect; forwards live events
  └─ mock rules                 CHECK_MOCK / SET_MOCK / REMOVE_MOCK / GET_MOCKS

Panel  (devtools/panel/)
  ├─ panel.js                   main controller: graph, map, table, sidebar, filters
  ├─ graph.js                   3D force-directed canvas renderer
  ├─ worldmap.js                equirectangular canvas map (TopoJSON country outlines)
  ├─ geoip-data.js              offline IP→country lookup — generated binary, no API calls
  └─ annotations.js             AI anomaly detection

Installation

No build step required. Load the folder directly into Chrome.

1

Clone the repository

git clone https://github.com/forge-veil/netphantom.git

2

Open the Extensions page

Navigate to chrome://extensions and enable Developer mode with the toggle in the top-right corner.

3

Load unpacked

Click Load unpacked and select the cloned repo folder.

4

Open DevTools

Press F12 on any tab and select the NetPhantom panel. Browse normally and watch the graph build itself.


Refresh the geo database (optional)

The bundled geoip-data.js was generated from DB-IP Lite at release time. To update it to the latest monthly snapshot: 

node tools/build-geoip.js

Privacy

NetPhantom is designed to be completely offline.

No telemetry No analytics, no crash reporting, no usage pings. Zero outbound traffic from the extension itself.
Offline geolocation Server IPs are resolved to countries using a bundled DB-IP Lite binary. No geo API calls are made.
Local-only data All captured request data stays in the panel's in-memory state and is cleared when you close DevTools.
Map topology Country outlines are fetched from jsDelivr (world-atlas TopoJSON) on first map open. This is the only outbound request NetPhantom makes.

Permissions explained

NetPhantom requests the minimum permissions needed.

PermissionWhy it's needed
webRequestCache real HTTP status codes before the DevTools event fires (fixes a Chrome bug where CORS-blocked requests report status 0).
tabsAssociate each DevTools panel with the correct inspected tab ID.
scriptingInject the content script that wraps fetch / XHR / WebSocket.
declarativeNetRequestReturn mocked responses for requests matching a user-defined URL pattern.
storagePersist mock rules so they survive a service worker restart.
<all_urls>Capture requests to any origin — necessary for a network inspector.
